# 一、全球食物数据库系统设计（Global Food Database）

1. 数据结构设计：

* 支持营养（宏量 + 微量）
* 支持食物分类（category / subCategory）
* 支持加工属性（processed / cookingMethod）
* 支持饮食文化（cuisine / region）
* 支持健康标签（low_carb / high_protein / diabetic_friendly）
* 支持替代关系（substitution graph）

2. 数据组织方式：

* 如何设计 Food Knowledge Graph（食物-营养-人群-疾病关系）

3. 数据来源与融合：

* USDA / 中国食物成分表 / OpenFoodFacts
* 如何做数据清洗、标准化、去重
* 如何通过 AI 自动补全缺失字段（如 GI、satiety）

4. 数据质量体系：

* 数据置信度（confidence）
* 数据来源分级（official / estimated / ai）
* 数据更新与版本控制策略

# 二、多维营养评分系统（Nutrition Scoring Engine）

请设计一个“多目标优化评分模型”，用于评价食物和餐单的健康性，要求：

1. 基础评分维度：

* 热量合理性（calorie alignment）
* 宏量营养（protein / fat / carbs balance）
* 微量营养（vitamins / minerals）
* 升糖指数（GI / GL）
* 饱腹感（satiety）
* 加工度（NOVA 或 processed level）

2. 高级评分维度：

* 多样性（food diversity）
* 长期健康（心血管 / 代谢风险）
* 用户目标（减脂 / 增肌 / 控糖）
* 饮食文化适配（中餐 / 地中海 / 日式）

3. 个性化权重系统：

* 如何根据用户目标动态调整权重
* 如何构建 scoring function（公式 or ML）

4. 输出：

* 单食物评分
* 单餐评分
* 每日评分
* 每周评分（趋势）

# 三、推荐引擎系统（Recommendation Engine）

请设计一个结合规则 + AI 的推荐系统，要求：

1. 推荐流程：

* 用户画像（健康 / 偏好 /限制）

* 组合生成（meal planning）
* 多目标排序（ranking）

2. 推荐策略：

* Rule-based（营养约束）
* Collaborative Filtering（用户偏好）
* Content-based（食物属性）
* Knowledge Graph 推理
* LLM 生成（meal plan）

3. 关键优化目标：

* 营养达标（硬约束）
* 多样性（避免重复）
* 可执行性（食材易获取）
* 用户满意度（acceptance rate）

4. 强化学习 / 反馈机制：

* 如何根据用户行为（点击 / 吃了 / 跳过）优化推荐
* 如何做长期饮食习惯建模
