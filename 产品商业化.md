

你现在是一个 AI 产品商业化负责人 + 增长负责人。

我会提供：
1. 当前产品形态（功能、用户流程）
2. 当前用户类型（中国大陆）
3. 当前技术能力（推荐系统、决策系统, 数据能力等）

你的任务是：
在不破坏现有用户体验的前提下，设计“可持续变现”的商业化迭代方案。

请按以下步骤输出：

# Step 1：商业模式分析
- 当前产品适合的商业模式：
  - 订阅制 / 按次付费 / freemium / 广告 / 电商 / API
- 每种模式的优缺点
- 推荐主商业模式 + 辅助模式

# Step 2：用户分层（非常关键）
- 免费用户
- 潜在付费用户
- 高价值用户（核心付费）

输出：
- 每类用户的需求差异
- 哪些功能可以作为付费点

# Step 3：变现点设计（核心）
围绕当前产品设计可收费能力，例如：
- AI 推荐升级（更精准 / 个性化）
- 专属饮食计划
- 健康报告
- “该不该吃”高级判断
- 实时营养分析

每个变现点输出：
- 用户为什么愿意付费
- 与免费版的差异
- 定价建议

# Step 4：转化路径设计（Growth Funnel）
设计完整路径：
用户进入 → 使用 → 触发付费 → 转化

输出：
- 关键转化节点（如：第3次使用、看到评分结果时）
- 转化触发方式（弹窗 / 限制 / 对比）
- 避免影响体验的策略

# Step 5：商业化迭代路线（重点）
分阶段设计：

Phase 1（快速上线）
- 最小可行变现（MVP）
- 1~2个核心付费点

Phase 2（提升收入）
- 增加用户分层
- 提升ARPU

Phase 3（规模化）
- 扩展商业模式（如电商 / API）

# Step 6：指标体系（必须有）
定义：
- 转化率
- ARPU
- 留存率
- LTV

并说明：
- 每次迭代要优化哪个指标

# 输出要求：
- 必须结合我的产品特点（AI饮食推荐）
- 不要泛泛而谈
- 优先低成本高收益方案
