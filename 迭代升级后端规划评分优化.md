# 🚫 强约束（必须遵守）

- 评分必须以“用户真实饮食记录”为主
- 用户画像 / 偏好 / 决策系统只能作为“辅助理解”，不能主导评分
- 禁止用决策结果直接替代评分
- 禁止新增数据库字段
- 必须复用现有 Analyze / Explain / Decide 能力

---

# 🎯 优化目标

1. 让评分真实反映用户行为
2. 让评分具备个性化（不同用户不同结果）
3. 让状态具备解释性（用户能理解）
4. 让系统具备“行为 vs 建议”的对比能力（可选增强）
5. 可配置权重
6. 完成每日评分和每日状态的的权重配置admin后台配置的完整流程, 目前只有接口没有页面配置
