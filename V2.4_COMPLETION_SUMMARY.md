## 🎯 V2.4 Architecture Overview

The Wuwei AI V2.4 system is a **3-phase architecture** for progressive i18n & admin analytics integration:

```
Phase 1: Scoring & Formatting (Foundation)
    ↓
Phase 2: Feedback Quality & I18n (Integration)
    ↓
Phase 3: Full I18n Coverage & Admin Dashboard (Dashboard & Analytics)
```

---

## 📋 Project Structure (Updated)

### Core Modules in `apps/api-server/src/`

#### **Phase 1: Scoring & Formatting**
- **`modules/scoring/`** - Nutrition scoring engine
  - `services/scoring.service.ts` - Core scoring logic
  - `services/decision-classifier.service.ts` - Decision classification
  - `services/coach-format.service.ts` - Output formatting

- **`modules/food/`** - Food analysis pipeline
  - `controllers/food-analysis.controller.ts` - Food analysis endpoints
  - `services/food-item.service.ts` - Food item management
  - `services/nutrition-compute.service.ts` - Nutrition calculation

#### **Phase 2: Feedback & I18n Foundation**
- **`modules/analysis-quality-feedback/`** - Quality feedback system
  - `services/analysis-quality-feedback.service.ts` - Feedback collection
  - `controllers/analysis-quality-feedback.controller.ts` - Feedback endpoints

- **`modules/i18n/`** - Internationalization layer
  - `i18n.service.ts` - Translation service
  - `extended-i18n.ts` - 60+ translation keys (EN, ZH, JA, KO, pt-BR)
  - Language support: English, Chinese (Simplified/Traditional), Japanese, Korean, Portuguese (Brazil)

- **`modules/food-analysis-report/`** - Analysis report generation
  - `controllers/food-analysis-report.controller.ts` - Report endpoints
  - Report generation with i18n support

#### **Phase 3: I18n Coverage & Admin Dashboard**
- **`modules/admin/quality-metrics/`** - Admin dashboard
  - `services/i18n-management.service.ts` - i18n administration
  - `controllers/admin-quality-metrics.controller.ts` - Admin endpoints
  - Quality metrics tracking and reporting
