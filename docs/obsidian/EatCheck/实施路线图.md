# 实施路线图

## Phase 1：现在，做中国兼容架构，不做中国正式上线

目标：不影响海外 PMF 和增长，同时让未来国内能力可插拔。

任务：

- [x] 创建 `RegionStrategyService`
- [x] 创建 `/api/app/capabilities`
- [x] 新增 routed `AiRuntimeService.chatRouted()` 入口
- [x] 将 food enrichment AI 调用接入可选 routed `AiRuntimeService` / AI runtime
- [x] 将 recipe generation AI 调用接入可选 routed `AiRuntimeService` / AI runtime
- [ ] 将其他业务 AI 调用逐步迁移到 routed `AiRuntimeService` / AI runtime
- [x] 给 `AiModelRouter` 增加 region/context 参数
- [x] 梳理 `AuthProviderRegistry`
- [x] 梳理 `BillingProvider`
- [x] Provider/model 配置增加 region 维度的最小路由支持
- [x] Region Strategy 增加 AI 模型路由配置：文本分析、视觉分析、fallback model
- [x] 建立最小合规开关：medical disclaimer、content moderation required、data residency required
- [x] Region Strategy 后台查看、更新、重置 region override
- [x] Region Strategy 后台管理页面查看、编辑、重置流程

验收标准：

- Global 用户现有登录、AI 分析、订阅不受影响
- 测试用户可以通过配置拿到 CN capability profile
- 业务模块不再新增直连 OpenAI/OpenRouter 代码
- 新 provider 接入只需要新增 adapter/provider 和配置，不需要改核心推荐逻辑

## Phase 2：产品稳定后，接入中国基础能力试点

前置条件：海外版 AI 分析、推荐、订阅闭环稳定。

任务：

- [ ] 接入 CN AI provider 的真实生产 key 与模型策略
- [ ] 手机号/微信登录按 CN profile 默认展示
- [ ] OSS/COS 或中国云存储试点
- [ ] 中国内容审核 provider 试点
- [ ] Android 中国支付接入方式复评，默认仍以 RevenueCat 能力抽象为主
- [ ] 设计中国节点部署方案，但不急于迁移核心数据

验收标准：

- 小规模中国测试用户可以完成登录、分析、订阅/支付沙箱流程
- CN 流量不会依赖 Firebase/OpenAI/OpenRouter 作为必需链路
- 日志和 usage records 能区分 region/provider/model

## Phase 3：真正中国化

前置条件：明确中国运营主体、发行渠道、合规预算和商业目标。

任务：

- [ ] ICP 备案与域名解析策略
- [ ] 中国云节点与数据驻留策略
- [ ] PIPL 个人信息与敏感信息处理规则
- [ ] 国内 Android 渠道发行
- [ ] 小程序或微信生态产品化
- [ ] 内容审核、投诉处理、用户协议、隐私政策中国版本

### 2026-05-13：后台模型配置与 AIProvider 串联检查

检查结论：

- 后台 Provider 管理写入 `providers`。
- 后台模型管理写入 `model_configs`，关联 `providerId`。
- Core AI routing 的 `AiModelRouter` 会读取 `clientCapabilityPermissions`、`model_configs`、`providers`，用于 routed `AiRuntimeService.chatRouted()`。
- Region Strategy 的 `aiModelRouting` 会指定业务功能使用的 provider / primary model / fallback model。

发现并修复：

- `AiModelRouter` 原本直接用 API 值 `text.generation` 查询 Prisma enum 字段，实际 Prisma Client 需要 `text_generation`，会导致 routed AI runtime 查不到模型。
- 已增加 API capability value 到 Prisma enum value 的映射。
- `RegionAiModelRoutingService` 原本只从 env 解析 provider key/baseUrl，没有读取后台 Provider/Model 配置。
- 已改为优先读取 `model_configs + providers`：
  - 命中后台配置时使用 `model.endpoint/customApiKey`，否则使用 provider `baseUrl/apiKey`。
  - 找不到后台模型配置时才回退 env，保证本地和灰度环境仍可用。
- `TextFoodAnalysisService`、`VisionApiClient`、`ImageNutritionFillService` 已适配异步模型路由。

### 2026-05-13：AI 管理配置链路审计与能力类型清理

检查范围：

- 后台 AI 服务管理：Provider、Model、Client Permission。
- Shared DTO：`CapabilityType`、模型/权限 DTO。
- 后端 Admin API：ProviderService、ModelService、PermissionService。
- Gateway：`CapabilityPermissionGuard`、`AiModelRouter`。
- Region Strategy：`RegionAiModelRoutingService`。
- Seed：Provider / Model 初始化。

链路结论：

- `CapabilityType` 的 canonical API 值统一以 `@ai-platform/shared` 为准。
