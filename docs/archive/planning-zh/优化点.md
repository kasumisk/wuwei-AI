## 一、整体架构评估（Architecture Diagnosis）

## 二、核心问题识别（Critical Issues）

从以下角度分析：

1. 数据建模问题（表结构 / 关系 / 冗余）
2. AI能力支持问题（embedding / 推荐 / 可解释性）
3. 扩展性问题（未来增加功能是否困难）
4. 性能隐患（尤其 PostgreSQL + 向量）
5. 数据一致性 & 来源可信度问题

要求：

👉 区分：
- “现在就必须解决的问题”
- “可以后期再处理的问题”

## 三、复杂度评估（Complexity Audit）

请分析：

  - 必须保留（核心能力）

特别关注：

- JSON字段
- 多表拆分程度
- AI相关字段设计

## 四、架构方向判断（最重要）

- 选择理由
