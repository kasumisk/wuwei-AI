# 七、问题判断标准

你必须判断以下问题是否真实存在：

## 推荐质量

- 是否推荐过敏食物
- 是否推荐用户明确避免的食物
- 是否和健康状况冲突
- 是否和饮食限制冲突
- 是否餐次不合理
- 是否场景不合理
- 是否地区不合理
- 是否季节不合理
- 是否预算不合理
- 是否烹饪难度不合理
- 是否设备不可行
- 是否重复度过高
- 是否解释和结果不一致

## 数据问题

- foodRegionalInfo 是否缺失
- allergens 是否缺失或格式不一致
- mainIngredient / foodGroup / category 是否不规范
- priceMin / priceMax / currencyCode / priceUnit 是否异常
- monthWeights 是否非法
- cuisine / source / availability 是否不足
- 用户画像是否未被正确读取

## 逻辑问题

- 权重是否被覆盖
- factor 顺序是否异常
- multiplier / additive 是否异常
- fallback 是否过度放宽
- min candidates 是否破坏硬约束
- cache key 是否缺 region / timezone / user dimension
- scene resolver 是否误判
- channel normalize 是否缺失

## AI 输出问题

- 结构是否稳定
- explanation 是否和最终推荐一致
- 是否出现不存在字段
- 是否把健康建议说成医疗诊断
- 是否输出语言和 locale 不一致

# 八、修复原则

每个问题按以下优先级修：

1. 数据字段错误：先修 seed / migration / 数据清洗 / 校验
2. 硬约束错误：优先修过滤和 health modifier
3. 推荐不合理：修 scoring factor / recall / assembly
4. 解释不一致：修 explanation mapping
5. 性能问题：只修明确瓶颈，不做大重构
6. 可维护性问题：只处理会导致 bug 的部分

禁止把业务规则简单删除来解决问题。
