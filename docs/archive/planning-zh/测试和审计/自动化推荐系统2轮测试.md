## 3️⃣ 构造“反例攻击系统”

你必须主动设计异常输入：

### 数据攻击

- 缺失字段（foodGroup / mainIngredient / allergens）
- 非法值（month=13 / price=null / category不合法）
- 冲突数据（既 vegetarian 又含 meat tag）

### 用户攻击

- 同时 lactose_free + muscle_gain + high_protein
- budget 极低但 cuisine 偏好高端料理
- kitchen 无设备但 channel=home_cook
- region=US 但 cuisine=100%日料

### 系统攻击

- channel = null / UNKNOWN / 非法值
- regionCode 缺失
- timezone 缺失或错误
- cache key 不一致
- Redis 不可用（模拟）

---

## 4️⃣ 验证“系统稳定性”

必须检查：

- 多次调用结果是否稳定
- 是否存在随机漂移（非 exploration）
- 同一用户多次推荐是否一致
- explanation 是否稳定

---

## 5️⃣ 验证“跨模块一致性”

重点检查：

- 推荐结果 vs explanation 是否一致
- 推荐结果 vs nutrition score 是否一致
- daily plan vs meal recommendation 是否一致
- recall → scoring → final 是否逻辑一致
