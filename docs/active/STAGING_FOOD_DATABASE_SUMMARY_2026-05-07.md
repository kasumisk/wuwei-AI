# Staging 食物库数据汇总（2026-05-07）

## 文档目的

- 汇总 `staging` 环境食物库三张核心表的数据规模与覆盖情况。
- 三张核心表分别为：
  - `foods`：食物主表
  - `food_regional_info`：食物地区信息表
  - `food_translations`：食物翻译表
- 为后续补翻译、修异常 locale、补细粒度地区数据提供依据。

## 统计口径说明

- 统计时间：`2026-05-07`
- 统计环境：`staging`
- 数据库类型：PostgreSQL
- 查询方式：直接连接 `apps/api-server/.env.staging` 指向的 staging 数据库执行 SQL 汇总
- 说明：
  - `foods_with_regional` 表示在 `food_regional_info` 中至少存在 1 行对应记录的食物数
  - `foods_with_translation` 表示在 `food_translations` 中至少存在 1 行对应记录的食物数
  - `coverage_pct` 表示覆盖率，计算公式为 `已覆盖食物数 / 总食物数 * 100%`
  - `avg_rows_per_food` / `avg_locales_per_food` 仅针对已覆盖食物计算平均值

## 一、总体结论

- `foods` 主表共有 `9580` 条食物数据。
- `food_regional_info` 共有 `57480` 条，覆盖 `9580` 个食物，地区覆盖率为 `100%`。
- `food_translations` 共有 `30969` 条，覆盖 `5161` 个食物，翻译覆盖率为 `53.87%`。
- 当前地区数据是标准化的 `6` 国模板覆盖，每个食物固定 `6` 条国家级地区信息。
- 当前翻译数据主要覆盖 `draft` 食物，`active` 食物翻译覆盖明显偏低。

- 多出来的第 `7` 个 locale 即异常值 `AU`。

## 八、异常点与风险说明

### 8.1 异常 locale：`AU`

- 当前在 `food_translations.locale` 中发现 `AU` 共 `3` 条。
- `AU` 更像国家码，不是标准语言区域码。
- 这类数据会影响：
  - 前端 locale 匹配
  - 翻译覆盖统计
  - 后续 i18n 清洗脚本

### 8.2 `active` 食物翻译覆盖过低

- `active` 食物共有 `5195` 条，但只有 `1441` 条有翻译。
- 覆盖率仅 `27.74%`。
- 若线上展示依赖翻译表，本项会直接影响多语言用户体验。

### 8.3 地区数据缺少 region/city 粒度

- 表结构支持 `country_code + region_code + city_code`。
- 但 staging 当前只有国家级记录，没有省/州/城市级记录。
- 这意味着地区适配能力还停留在国家级模板阶段。

### 8.4 翻译来源为空的记录较多

- `source is null` 的翻译共有 `8652` 条。
- 同时它们的 `review_status = PENDING`。
- 这通常说明：
  - 历史导入数据未补 source
  - 批量生成流程中断
  - 数据已写入，但未完成审核/归档流程
