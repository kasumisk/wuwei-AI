# 🎯 优化目标（必须覆盖）

## 1. 订阅体系优化（核心商业能力）
目标：提升转化率 + ARPU

需要设计：
- 用户权限分层（免费 / 订阅 / 高级）
- 功能分级：
  - 免费：基础推荐 + 简单分析
  - 订阅：深度分析 + 个性化推荐 + 历史记录
- 订阅控制机制：
  - 按功能限制（如分析次数 / 高级解释）
  - 按能力限制（推荐精度/策略）
- 付费触发点设计（在用户“犹豫时”触发）

输出：
- 订阅模型设计（字段 / 权限控制）
- 后端实现方式（guard / middleware / service）
- 如何与推荐系统、分析系统联动

## 2. 食物分析能力（重点模块）
目标：从“输入食物” → “结构化分析结果”

需要支持两种输入：

### （1）文本分析（用户输入食物名称/描述）
- 如：鸡胸肉 / 牛肉面 / 沙拉
- 处理流程：
  - 标准化（映射到食物库）
  - 营养分析
  - 推荐/决策输出（该不该吃）

---

### （2）图片分析（用户拍照）
- 如：外卖 / 实拍食物
- 处理流程：
  - 图片识别（识别食物类别）
  - 食物拆解（多食物组合）
  - 估算营养（不需要100%准确，但要合理）
  - 输出分析结果

⚠️ 要求：
- 文本与图片必须分开处理链路（不能混用）
- 但最终统一输出结构

---

## 3. 食物数据入库（非常关键）
目标：让系统“越用越强”

需要设计：
- 分析后的数据是否入库
- 如何入库：
  - 标准食物（匹配已有库）
  - 新食物（创建新记录）
- 数据结构设计：
  - 原始输入（文本/图片）
  - 解析结果
  - 营养数据
  - 置信度

需要考虑：
- 数据去重
- 数据质量控制
- 是否人工审核（可选）

## 4. 统一分析结果结构（关键）
无论文本还是图片，输出统一格式：

例如：
- 食物名称
- 营养信息
- 健康评分
- 是否建议食用
- 替代建议
- 解释信息
